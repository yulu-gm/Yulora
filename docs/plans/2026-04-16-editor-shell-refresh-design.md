# Editor Shell Refresh Design

## 目标

在不改变现有编辑器内核、主题系统职责边界和 Markdown 编辑语义的前提下，把当前偏 MVP 的页面壳升级为更像成熟桌面编辑器的前端界面。

本轮设计重点不是颜色、字号和具体主题风格，而是：
- 页面布局
- 信息层级
- 编辑器壳层的桌面应用感
- 设置交互方式
- 状态信息展示方式

## 非目标

本轮明确不做：
- 大纲、最近文件、搜索替换、导出等新功能
- 编辑器内核重构
- Markdown 渲染语义调整
- IME / undo redo / autosave 机制改写
- 用固定颜色和字体覆盖主题系统
- 常驻宽侧栏或伪功能入口

## 总体结构

主界面采用三层结构：

1. `rail`
   最左侧窄轨道，负责提供应用壳感和少量稳定入口。

2. `workspace`
   中间主工作区，始终是视觉核心，承载空态、文档头部、文档画布与状态条。

3. `settings drawer`
   从左侧滑入的设置抽屉，覆盖 `rail` 右侧和部分 `workspace`，而不是整页切换。

该结构的目标是：
- 默认仍保持单栏写作体验
- 让界面更像桌面应用而不是网页 demo
- 为未来大纲 / 最近文件 / 搜索预留自然落位
- 避免在功能未接入前出现大面积空侧栏

## Rail 设计

`rail` 是窄轨道，不是宽侧栏。

包含内容：
- 顶部品牌区
- 少量占位入口或锚点
- 底部固定设置入口

设计约束：
- 轨道宽度稳定，不随内容波动
- 不伪装未实现功能为可点击能力
- 占位入口应尽量中性，不制造“半成品侧栏”观感
- 设置入口固定在轨道底部，形成稳定肌肉记忆

## Workspace 设计

`workspace` 是主编辑区，采用“壳层自适应 + 画布舒适宽度上限”的布局策略。

### 宽度策略

- 工作区跟随窗口尺寸变化
- 文档画布可以随窗口变宽而增大
- 画布存在阅读舒适上限，避免无限拉伸
- 窄窗口下优先压缩留白，再压缩画布宽度
- 保持最低可编辑宽度，避免压成网页式窄栏

### 已打开文档态的结构

文档打开后，`workspace` 分成三个纵向区块：

1. `document header`
   展示文档身份信息

2. `editor canvas`
   展示主编辑器，保持视觉中心地位

3. `status strip`
   展示持续变化但优先级较低的信息

## 空态设计

空态不再使用大面积居中 hero，而是使用位于 `workspace` 内的安静起始面板。

内容只放当前真实可用的信息：
- 打开一个 Markdown 文档开始编辑
- 本地优先 / Markdown 保真
- `Ctrl/Cmd + O` 提示

空态要求：
- 保留 `rail`
- 避免 marketing landing page 感
- 不摆未实现的最近文件 / 大纲功能

## 文档头部设计

文档头部是“文档身份区”，不承担全部运行状态。

包含内容：
- 文档名
- 文档路径
- 轻量 dirty 标记或未保存提示

不放或弱化的内容：
- 自动保存中的动态文案
- 字数
- 平台桥接信息

这样可以让顶部保持安静，避免成为状态公告栏。

## 状态条设计

状态条用于承接会持续变化的信息。

本轮至少承载：
- 保存状态
  - `All changes saved`
  - `Unsaved changes`
  - `Saving...`
  - `Autosaving...`
  - 失败错误态
- 文档统计
  - `字数`
- 次级运行信息
  - `Bridge: platform` 这类开发态信息下沉到最右侧并弱化

设计原则：
- 抬头看“当前编辑什么”
- 低头看“当前保存到哪一步、写了多少”

## 设置抽屉设计

设置从“整页切换”改为“左侧抽屉”。

### 交互模型

- 由 `rail` 底部设置入口触发
- 从左侧滑入
- 覆盖 `rail` 右侧和部分 `workspace`
- 点击遮罩、点击关闭按钮、按 `Esc` 都可关闭

### 视觉模型

- 抽屉允许轻度毛玻璃
- 毛玻璃只用于抽屉层，不蔓延到主画布
- 抽屉表单区仍需有清晰边界和高可读性
- 背景工作区继续可见，但做轻度退让

### 内容模型

设置内容延续现有分组：
- 主题
- 排版
- 自动保存
- 最近文件（保留禁用占位）

但展示从“页面主体”改为“抽屉内容”。

## 动效原则

动效存在，但必须克制。

允许的动效：
- 设置抽屉短时长侧滑进入 / 退出
- 主工作区轻度位移、暗化或模糊
- 设置入口 hover 的轻微交互反馈

避免的动效：
- 大幅缩放
- 弹簧感过强的过渡
- 与写作工具气质不符的花哨动画

兼容要求：
- 必须支持 `prefers-reduced-motion`
- reduced motion 模式下应退化为近乎无动画

## 编辑器与抽屉的关系

设置抽屉是覆盖层，不是页面切换。

因此：
- 编辑器在抽屉打开时仍保持挂载
- 打开抽屉会导致编辑器失焦，这应沿用现有 blur -> autosave 语义
- 抽屉开合不得重建编辑器实例
- 抽屉关闭后应恢复到抽屉打开前的焦点语义
  - 若打开前焦点在编辑器内，则关闭后把焦点还给编辑器
  - 若打开前焦点不在编辑器，则回到触发按钮

## 文档统计

本轮状态区只承载 `字数`，不同时引入 `词数`、阅读时长或选区统计，避免范围膨胀。

原因：
- 中文语境下“字数”是更自然的主指标
- 可以避免在首轮就引入复杂的多语言词法分词策略
- 已能满足用户对“文档字数等信息”的核心诉求

如后续需要，再扩展：
- `字符数`
- `词数`
- 选区统计
- 阅读时长

## 响应式规则

### 宽窗口

- 显示完整 rail
- 工作区保留较宽留白
- 画布保持舒适阅读上限
- 抽屉宽度为中等桌面尺寸

### 中等窗口

- 保留 rail
- 压缩留白
- 抽屉按比例变窄

### 窄窗口

- 仍保留轨道语义，但可以进一步压缩内部内容
- 抽屉接近全高、接近全宽的左侧覆盖层
- 不切换成移动端底部导航模式

## 验收重点

完成后应满足：
- 空态和文档态都更像桌面编辑器而不是 demo 卡片
- 左侧轨道建立应用壳感，但不抢主画布注意力
- 文档画布随窗口变化自然缩放，并保持舒适宽度上限
- 保存状态、自动保存和字数信息有稳定落位
- 设置改成左侧抽屉，支持滑入、遮罩关闭、Esc 关闭
- 抽屉开合不破坏编辑器挂载、滚动位置、选择位置和 autosave 行为
- 主题切换后布局仍稳定成立

## 风险与约束

- 若为抽屉开合重建编辑器，会直接破坏输入、选择和 autosave 语义
- 若动画和毛玻璃使用过重，会偏离“写作工具”气质
- 若把结构决策写死到主题层，会与当前主题系统职责冲突
- 若引入过多占位入口，会让界面显得像未完成的 Obsidian 仿品
