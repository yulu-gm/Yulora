# TASK-035 Execution Handoff

## 改了什么

- 在 `src/renderer/code-editor.ts` 内加入 composition guard
- 组合输入期间仍然透传 `onChange`，保证编辑文本与 autosave 链路不变
- 组合输入期间暂停 `parseBlockMap()` 和 active-block 广播，等 `compositionend` 后统一 flush 一次
- 修正 `src/renderer/code-editor-view.tsx` 只要 `initialContent` 跟随 autosave 成功回写就触发 `replaceDocument()` 的问题，避免保存后重建 editor state 导致光标跳到文首
- 在 `src/renderer/code-editor.test.ts` 增加段落、标题、列表三类 IME regression tests
- 新增 `src/renderer/code-editor-view.test.tsx`，锁定“同一 `loadRevision` 下 autosave 回写内容不应重载编辑器”的回归
- 同步更新 backlog、进度、决策日志和测试记录

## 人工验收草稿

1. 运行 `npm run dev`
2. 切换到中文输入法
3. 在普通段落中连续输入中文，确认组合期间不丢字、不跳光标
4. 在 `# ` 标题后连续输入中文，确认组合期间不丢字、不跳光标
5. 在 `- ` 列表项后连续输入中文，确认组合期间不丢字、不跳光标
6. 失焦一次，确认 autosave 行为仍正常

## 已知风险与未做项

- 当前 guard 只保护 `block map -> active block -> 消费方` 这条派生状态链；未来真正引入 decorations / source-render toggle 时，必须复用同一 guard，而不是绕过它直接更新视图
- 未覆盖引用块、代码块、多光标、真实 Windows/macOS IME 事件差异
- 本轮未做 Playwright 或桌面级自动化 IME 验证，仍需 acceptance 阶段补人工验证证据
