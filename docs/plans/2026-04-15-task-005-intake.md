# TASK-005 Intake

Task: `TASK-005`
Goal: Implement full autosave for already opened Markdown documents, including idle-triggered autosave, blur-triggered autosave, distinct autosave/manual-save status, and failure-safe behavior that preserves in-memory edits.

In scope:
- autosave for documents that already have a file path
- idle autosave after typing pauses
- blur autosave from the editor boundary
- save-state distinction between manual save and autosave
- replay gating so only one write is in flight at a time
- regression-safe tests for renderer state, editor blur, and autosave orchestration

Out of scope:
- untitled new-document save flows
- crash recovery
- window-close interception
- recent files
- Markdown block rendering or IME-specific work

Primary files/modules:
- `src/renderer/document-state.ts`
- `src/renderer/document-state.test.ts`
- `src/renderer/code-editor.ts`
- `src/renderer/code-editor-view.tsx`
- `src/renderer/code-editor.test.ts`
- `src/renderer/App.tsx`
- `src/renderer/app.autosave.test.ts`

Verification:
- `npm run test -- src/renderer/document-state.test.ts`
- `npm run test -- src/renderer/code-editor.test.ts`
- `npm run test -- src/renderer/app.autosave.test.ts`
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`

Manual acceptance:
- open an existing `.md` file
- edit, pause, and confirm autosave writes to disk
- edit again, blur the editor, and confirm autosave runs
- simulate a save failure and confirm the document stays dirty and content remains visible
- confirm manual `Save` and `Save As` still work
