# TASK-028 Debug UI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Turn the test workbench placeholders into a real-time debug surface that shows scenario progress, step state, recent events, and terminal failure or interrupt reasons.

**Architecture:** Keep execution local to the renderer. The workbench calls `runScenario()` directly, folds `RunnerEvent` objects into a small UI-facing run-state model, and renders that model across the existing workbench panels. No new main/preload bridge is introduced in this task.

**Tech Stack:** React, TypeScript, Electron renderer, Vitest
