# TASK-034 Execution Handoff

## 改了什么

- 在 `packages/markdown-engine/src/` 建立了完整 inline AST、canonical `parseMarkdownDocument()` 入口，以及本地 `strikethrough` extension
- 保留现有 top-level `blockMap` 作为块级语义边界，`parseBlockMap()` 继续作为兼容入口
- 在 `packages/editor-core/src/` 引入 `MarkdownDocumentCache`，把 derived state、signature 与 inactive decorations 迁到 MarkdownDocument
- 新增 AST-to-decoration flattening，把 `**bold**`、`*italic*`、`` `code` ``、`~~strike~~` 以及常见嵌套渲染到非激活态 block
- 在 `src/renderer/code-editor.ts` 改为使用 `parseMarkdownDocument`，并在 `src/renderer/styles.css` 中补上 inline marker / content 样式
- 在 `src/renderer/code-editor.test.ts`、`packages/editor-core/src/decorations/block-decorations.test.ts`、`packages/editor-core/src/derived-state/inactive-block-decorations.test.ts` 等位置补齐回归测试

## 人工验收草稿

1. 运行 `npm run dev`
2. 输入 `**bold**`、`*italic*`、`` `code` ``、`~~strike~~`，再把光标切到其他 block
3. 确认非激活态隐藏 Markdown markers，并显示粗体、斜体、行内代码、删除线样式
4. 输入 `***both***` 与 `~~**mix**~~`，确认嵌套样式同时成立
5. 在 heading、list、blockquote 内重复上述验证，确认同样成立
6. 把光标移回原 block，确认完整 Markdown 源码恢复
7. 切换到中文输入法，在含行内格式的文档中做一次 composition 输入，确认 composition 期间不抖动，结束后只 flush 一次

## 已知风险与未做项

- link/image 本轮只保证不破坏 label/alt children 的 inline decorations，没有做专门视觉替换
- reference-style links/images 与更复杂的 Markdown 歧义边界仍未完整纳入测试
- 本地 `strikethrough` extension 依赖 micromark 事件重写，后续升级 micromark 时需要重点回归
- 本轮尚未单独记录桌面人工验收，因此 `docs/progress.md` 先记为 `DEV_DONE`，未提升到 `CLOSED`
