# TASK-012 Intake

Task: TASK-012
Goal: 在保持 Markdown 源码为唯一事实来源的前提下，为无序列表、有序列表和任务列表提供稳定的非激活态渲染，并补齐 Enter 续项与空项退出语义。
In scope:
- 为列表块补足渲染所需的 block map metadata，至少覆盖无序列表、有序列表、任务列表和嵌套列表项的显示判断
- 在 `CodeMirror` 的非激活态装饰管线里增加列表渲染，保持激活块继续显示 Markdown 源码
- 定义并实现 Enter 续项、任务列表续项、空项退出列表的最小可预测规则
- 为 block map、编辑器渲染和 Enter 行为补测试，覆盖关键嵌套回归场景
Out of scope:
- 鼠标点击切换任务列表勾选状态
- 回写 Markdown 以外的新数据模型或富文本状态
- 引入新的编辑器内核、独立渲染层或 React 侧 Markdown 解析
- 超出 TASK-012 的引用块、链接、图片或代码块渲染
Landing area:
- `packages/markdown-engine/src/`：扩展列表 block metadata 与解析测试
- `packages/editor-core/src/`：必要时补列表激活态解析辅助
- `src/renderer/code-editor.ts`：列表非激活态装饰与 Enter 命令
- `src/renderer/code-editor.test.ts`：列表渲染和 Enter/退出语义测试
- `src/renderer/styles.css`：列表非激活态样式
Acceptance:
- `- item`、`1. item`、`- [ ] item` 在非激活态具备稳定、可区分的列表显示
- 光标进入列表块时保持 Markdown 源码可直接编辑
- 在非空列表项按 Enter 可续项，保留合适的 marker / task marker
- 在空列表项按 Enter 可退出当前列表层级
- 嵌套列表与任务列表的关键回归场景有自动化测试覆盖
Verification:
- `npm run test -- packages/markdown-engine/src/parse-block-map.test.ts`
- `npm run test -- src/renderer/code-editor.test.ts`
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
Risks:
- 直接触及 P0 相关的光标映射、undo/redo 语义与 Markdown round-trip 保真
- Enter 行为若实现位置不对，容易和 IME composition guard 冲突
- 嵌套列表的 metadata 不足时，非激活态渲染与空项退出规则都可能失真
- 需要确保 macOS / Windows 的键盘行为一致
