# TASK-005 Handoff

改了什么：
- 在 renderer shell 中实现 autosave 调度，支持 idle autosave、blur autosave、手动保存优先级和 in-flight save 后的单次 replay autosave
- 扩展保存状态机，区分 `manual-saving` 与 `autosaving`
- 给 CodeMirror 包装层补充 blur 事件回传，保持 editor 只负责事件，不负责持久化
- 新增 autosave orchestration 测试，并补齐 vitest 对 TSX/renderer test typing 的编译支持

人工验收草稿步骤：
1. 运行 `npm run dev`
2. 通过 `File > Open...` 打开一个已有 `.md` 文件
3. 编辑文本后停止输入约 1 秒，确认文件自动写回磁盘，界面状态回到 `All changes saved`
4. 再编辑一次文本并切走编辑器焦点，确认立即触发 autosave
5. 验证 `File > Save` 在有 pending autosave 时仍只执行一次保存
6. 模拟写入失败，确认界面显示 `Autosave failed. Changes are still in memory.`，并且当前文本与 dirty 状态都保留
7. 验证 `File > Save As...` 仍可正常切换当前路径

已知风险或未做项：
- 当前 blur 触发基于 editor 边界的 focusout 透传，后续如果引入更复杂的 editor 内部焦点结构，需要再次确认不会误判内部焦点切换
- 未覆盖未命名新文档 autosave、崩溃恢复、窗口关闭拦截
