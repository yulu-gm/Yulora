Task: TASK-015
Goal: 支持从剪贴板粘贴图片到当前 Markdown 文档时，把图片写入本地 `assets` 目录、插入相对路径 Markdown，并让编辑器中的图片在激活态与非激活态都可见；激活态额外在图片上方显示对应 Markdown 源码，保持可直接编辑。
In scope:
- 定义图片资源目录、文件名与冲突处理策略
- 在 main / preload 建立受限的“从剪贴板导入图片”桥接
- 在 renderer / editor-core 拦截图片粘贴并插入 Markdown
- 为 Markdown image inline AST 增加图片渲染能力
- 激活态显示“源码在上、图片在下”，非激活态显示稳定图片预览
- 失败、非图片剪贴板、超限与不支持格式场景的显式处理
Out of scope:
- TASK-016 的拖放导入
- 远程图片下载、本地图片重命名 UI、图片尺寸调整 UI
- reference-style image 完整支持
- HTML / PDF 导出中的图片路径转换
Landing area:
- src/main/
- src/preload/
- src/renderer/
- packages/editor-core/
- docs/
Acceptance:
- 粘贴 PNG / JPG 后成功写入 `assets`
- 编辑器插入相对路径 Markdown
- 图片在编辑器中可见
- 激活图片所在 block 时，Markdown 源码仍可直接编辑，且图片不消失
- 重名冲突处理正确
