# TASK-014 链接显示与编辑 intake

Task: TASK-014 链接显示与编辑

Goal: 让 Markdown 链接在非激活态显示为可读链接文本，在激活态仍显示原始 Markdown 语法，并通过受限桥接打开系统浏览器。

In scope:
- 基于 `markdown-engine` 已有 inline link AST 生成非激活态链接视觉装饰。
- 在 editor-core 中提供链接点击与键盘打开语义，同时保留点击回到源码编辑的路径。
- 在 preload / main 之间新增只接收链接 URL 的 IPC 桥接，由 main 进程校验协议后调用系统浏览器。
- 补链接显示、交互桥接和 main/preload wiring 测试。

Out of scope:
- 不实现相对 Markdown 链接的文件跳转。
- 不新增链接编辑弹窗、悬浮工具栏或富文本 URL 面板。
- 不改变 `markdown-engine` 现有 link AST 契约，除非测试暴露解析缺口。

Landing area:
- `packages/editor-core/src/decorations/inline-decorations.ts`
- `packages/editor-core/src/extensions/markdown.ts`
- `src/renderer/code-editor.ts`
- `src/shared/`
- `src/preload/preload.ts`
- `src/main/main.ts`
- `src/renderer/styles/markdown-render.css`
- 相关 Vitest 测试与项目记录文档

Acceptance:
- 失焦或非激活态时，链接正文以链接样式显示，Markdown URL / 括号语法不作为正文视觉重点出现。
- 光标进入链接所在 block 后，完整 Markdown 源码恢复并可直接编辑。
- 用户可通过安全桥接把 `http:` / `https:` / `mailto:` 链接交给系统浏览器。

Verification:
- `npm.cmd run test -- packages/editor-core/src/decorations/block-decorations.test.ts packages/editor-core/src/extensions/markdown.test.ts src/renderer/code-editor.test.ts src/preload/preload.test.ts src/preload/preload.contract.test.ts src/main/main.test.ts`
- `npm.cmd run typecheck`
- `npm.cmd run lint`
- `npm.cmd run build`

Risks:
- 链接点击不能破坏既有 active block 切换、隐藏 marker 光标归一和 IME composition guard。
- main/preload 桥接必须保持白名单协议，不把任意 Node / shell 能力暴露给 renderer。
