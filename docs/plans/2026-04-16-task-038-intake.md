# TASK-038 Intake

Task: TASK-038

Goal: 为当前项目增加一个本地 Windows 打包入口，让开发者能在现有仓库内通过单条命令直接生成可安装的 `.exe` 安装包。

In scope:
- 选择并接入最小可用的 Windows 打包方案
- 新增 `npm run package:win` 一类的打包入口
- 复用现有 `build` 产物生成 NSIS `.exe`
- 约束安装包仅包含运行时必需文件
- 补充最小应用元信息与打包说明

Out of scope:
- macOS 打包
- 自动更新
- 代码签名与证书
- 正式品牌图标与安装器深度定制
- CI 发布流程

Landing area:
- `package.json`
- `package-lock.json`
- 打包配置文件（根目录）
- `src/main/package-scripts.test.ts`
- 打包说明文档
- 必要的 backlog / progress / decision-log / test-report 记录

Acceptance:
- 仓库存在明确的 Windows 打包命令入口
- 在 Windows 本机可生成可安装的 `.exe`
- 安装包可启动当前 Electron 应用
- 文档明确说明本地打包方式与当前限制

Verification:
- `npm run test -- src/main/package-scripts.test.ts`
- `npm run build`
- `npm run package:win`
- Windows 本地最小人工冒烟安装与启动

Risks:
- 打包范围配置过宽，导致源码、测试或文档被打入安装包
- 图标资源尚未就绪，可能需要先接受默认图标占位
- 如果把打包入口混入现有 `build`，会增加构建与分发职责耦合
