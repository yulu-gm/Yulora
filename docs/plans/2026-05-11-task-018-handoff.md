# TASK-018 查找替换 handoff

## 改了什么

- 新增 renderer 查找替换面板，入口在左侧 rail，并支持 `Ctrl/Command+F` 打开。
- `CodeEditorController` 接入 CodeMirror search state，支持设置 query、匹配计数、上一项 / 下一项、替换当前、替换全部和清理 query。
- 通过隐藏 CodeMirror search panel 启用 `cm-searchMatch` / `cm-searchMatch-selected` 高亮，不渲染 CodeMirror 默认面板。
- `CodeEditorView` 透传查找替换句柄，`WorkspaceShell` 只维护临时输入状态并委托给编辑器控制器。
- 补充控制器与 WorkspaceShell 回归测试，覆盖查找导航、替换当前、替换全部和 undo history。
- 同步 backlog、progress、design、decision log、test case、test report 和任务总结。

## 人工验收草稿

1. 打开 FishMark，并准备一个包含 `alpha beta` 与 `Beta alpha` 的 Markdown 文档。
2. 点击左侧 rail 的查找 icon，或按 `Ctrl+F`（macOS 为 `Command+F`）。
3. 输入 `beta`，确认正文中两个匹配项被高亮，面板显示 `1 / 2`。
4. 点击下一项 / 上一项，确认当前匹配选区移动且文档内容不变化。
5. 在替换框输入 `gamma`，点击 `Replace`，确认只替换当前匹配。
6. 点击 `All`，确认剩余匹配全部替换。
7. 按 `Ctrl+Z`（macOS 为 `Command+Z`），确认上一步替换可以自然撤销。
8. 关闭查找替换面板，确认高亮清除，编辑器焦点回到正文。

## 已知风险或未做项

- 本轮只交付 literal、大小写不敏感的当前文档全文查找替换，不包含正则、大小写开关、whole-word、跨文件搜索或搜索历史。
- 还没有真实桌面人工验收记录，因此状态先同步为 `DEV_DONE`，等待 acceptance 阶段复核。
