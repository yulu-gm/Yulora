# TASK-006 execution handoff

Date: 2026-05-02

## 改了什么

- 新增最近文件共享 contract：IPC channel、`RecentFileEntry` / `RecentFilesSnapshot`、归一化与序列化规则。
- 新增 main 进程 `recent-files-service`，把最近文件持久化到 `<userData>/recent-files.json`，支持去重置顶、按 `recentFiles.maxEntries` 裁剪、清理失效路径和变更广播。
- 在成功打开、从路径打开、reload、保存和另存为后记录最近文件；最近文件写入失败只打 warning，不阻断打开 / 保存主链路。
- preload 暴露 `getRecentFiles`、`clearRecentFile`、`onRecentFilesChanged`，renderer 不获得 Node 文件系统能力。
- renderer 空工作区显示最近文件列表；点击后复用现有 workspace open flow，打开失败时清理该失效项。
- 设置页的最近文件上限从“待 TASK-006 开放”改为可编辑控制。

## 人工验收草稿

1. 启动应用并打开一个 `.md` 文件。
2. 回到空工作区，确认最近文件列表显示该文件名称和路径。
3. 点击最近文件项，确认该文档在当前 workspace 重新打开。
4. 在系统外部移动或删除一个最近文件路径，再点击该项，确认 FishMark 报错并从列表移除该项。
5. 打开设置中的 `最近文件`，把上限改为 `0`，确认列表隐藏；再恢复默认上限。

## 已知风险或未做项

- 本任务不做 workspace session restore，也不恢复上次打开的标签页集合。
- 最近文件只记录路径和显示名，不做固定、搜索、分组或历史版本。
- 真实桌面人工验收仍需在 dev app 中确认文件移动 / 删除后的报错与清理体验。
- 默认全量 `npm run test` 在当前环境出现 Vitest worker / timeout 阻塞；相关目标测试与失败文件单独复核均通过。
