# TASK-028 Intake

Task: TASK-028
Goal: 让测试工作台在同一窗口内实时显示场景执行进度、步骤状态、最近事件、失败原因和中断原因，作为 agent 和开发者排障的主承载面。
In scope:
- 在 `src/renderer/` 中把 `packages/test-harness` 的 runner 事件流接到 workbench UI
- 展示当前场景概览、当前步骤、每步状态与耗时
- 展示最近事件流，以及失败 / 中断时的原因摘要
- 保持 workbench-first 调试模型，不把 debug 信息下沉到 editor 测试窗口
Out of scope:
- 新增 main / preload 跨进程订阅协议
- CLI 统一入口与退出码
- visual-test 截图、baseline、diff 工件
- 新的真实场景执行器或复杂进程编排
Landing area:
- `src/renderer/App.tsx`
- `src/renderer/styles.css`
- `src/renderer/test-workbench.test.tsx`
- `packages/test-harness/src/runner.ts`
- `packages/test-harness/src/index.ts`
- `packages/test-harness/src/runner.test.ts`
Acceptance:
- workbench 可显示总步骤数、当前步骤和场景终态
- workbench 可显示每个步骤的状态与耗时
- 失败和中断时可看到原因、定位到步骤，并查看最近事件流
- 正常 editor 模式不受影响
Verification:
- `npm run test -- packages/test-harness/src/runner.test.ts`
- `npm run test -- src/renderer/test-workbench.test.tsx`
- `npm run lint`
- `npm run typecheck`
- `npm run build`
Risks:
- 若 renderer 直接拼接 runner 结果而没有稳定视图模型，workbench 状态会难以维护
- 若事件日志无限增长，长时间运行会污染 UI 和内存
- 若把调试状态耦合进 editor 窗口，会破坏测试工作台作为单一调试面的边界
