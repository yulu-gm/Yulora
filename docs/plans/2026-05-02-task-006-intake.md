## Task

TASK-006 最近文件

## Goal

记录最近成功打开或保存的 Markdown 文档路径，在 renderer 中展示最近文件入口，并支持点击重新打开与清理失效路径。

## In scope

- 定义最近文件列表的共享 contract、持久化结构和更新规则。
- main 进程在成功打开、保存、另存为后维护最近文件列表。
- preload 只暴露受限的最近文件读取 / 清理 API 与变更事件。
- renderer 在空工作区展示最近文件列表，并点击后复用现有 workspace open flow。
- 打开失效路径失败时，从最近文件列表中清理该项。
- 接通 `recentFiles.maxEntries` 上限配置，重复路径去重并移动到列表顶部。

## Out of scope

- 崩溃恢复、workspace session restore 或跨窗口完整会话恢复。
- 最近文件的搜索、分组、固定、历史版本或云同步。
- 改写 Markdown 保存格式、编辑模型或文件 watcher 语义。
- 新增 renderer 侧 Node 文件访问能力。

## Landing area

- `src/shared/`：最近文件共享类型、IPC channel 与 bridge contract。
- `src/main/`：最近文件持久化服务、IPC handlers、打开 / 保存成功后的更新。
- `src/preload/`：受限 bridge 方法与事件转发。
- `src/renderer/editor/`：空态最近文件 UI、点击打开、失效清理。
- `docs/`、`reports/task-summaries/`、`MVP_BACKLOG.md`：任务状态、测试记录和交接文档。

## Acceptance

- 可以显示最近文件列表。
- 点击最近文件后能重新打开。
- 失效路径可清理。
- 最近文件只记录成功打开或保存的 Markdown 文档。
- 重复路径不会生成重复项，而是更新顺序。
- `recentFiles.maxEntries` 为 0 时列表为空；超过上限时裁剪旧项。

## Risks

- 文件持久化必须继续由 main 进程负责，不能向 renderer 暴露 Node API。
- 打开最近文件前要 flush 当前活动 tab draft，避免切换文档丢编辑内容。
- 失效路径清理不能误删仍可打开的项；只在明确 open-from-path 返回错误时清理。
- 最近文件配置更新后要同步裁剪列表，避免偏好上限与列表展示互相打架。
