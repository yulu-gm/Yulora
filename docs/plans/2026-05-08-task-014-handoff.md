# TASK-014 链接显示与编辑 handoff

## 改了什么

- `editor-core` 基于现有 inline link AST 为非激活态链接添加可读 label 装饰，隐藏 `[]()` 与 destination 语法，并保留 label 内部 strong / emphasis / code 等子装饰。
- 新增链接打开交互：非激活态 `Mod-click` 打开链接，编辑态或源码范围内 `Mod-Enter` 打开光标所在链接；普通点击仍可回到源码编辑路径。
- `renderer` 将链接打开请求接入 `CodeEditorController` / `CodeEditorView` / `WorkspaceShell`，最终调用 `window.fishmark.openExternalLink()`。
- `preload` / `main` 新增 `fishmark:open-external-link` IPC；main 进程只允许 `http:`、`https:`、`mailto:` 后调用 `shell.openExternal()`。
- 补充 TASK-014 相关测试、test case、决策记录和 backlog/progress 状态。

## 人工验收草稿

1. 打开 FishMark，输入 `[FishMark](https://fishmark.app)` 和另一段普通文本。
2. 点击普通文本段落，让链接段落进入非激活态，确认链接显示为可读 label，URL 语法不突出显示。
3. 按住 `Ctrl`（macOS 为 `Command`）点击链接 label，确认系统浏览器打开目标 URL。
4. 把光标放回链接源码范围内，确认完整 Markdown 源码恢复并可编辑。
5. 在源码范围内按 `Ctrl+Enter`（macOS 为 `Command+Enter`），确认同样会打开系统浏览器。

## 已知风险或未做项

- 本轮只打开 `http:`、`https:`、`mailto:`，不处理相对 Markdown 链接的文件跳转。
- 本轮不提供链接编辑弹窗、悬浮工具栏或链接预览 tooltip。
