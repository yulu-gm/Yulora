# TASK-013 Intake

Task: `TASK-013`
Goal: 在保持 Markdown 源码为唯一事实来源的前提下，为引用块交付稳定的非激活态显示与激活态源码编辑切换，让引用块在阅读时更接近渲染态、在编辑时仍可直接操作 `>` 源码。

In scope:
- 只推进 `TASK-013`，不串做链接、代码块、行内格式或 IME 专项修复
- 复用 `TASK-009` 的 active block 状态与 `TASK-010/011/012` 的 CodeMirror decoration 派生链，为 top-level `blockquote` 增加非激活态显示策略
- 弱化非激活引用块中的 `>` 前缀，并给引用块补充整块级的可读性样式
- 保证光标进入引用块时稳定回到 Markdown 源码态，允许直接编辑 `>` 前缀与正文
- 为引用块补充自动化回归测试，覆盖装饰切换、激活态恢复与 composition guard 下的派生状态 flush

Out of scope:
- blockquote 内部嵌套 list / code block / heading 的额外渲染语义
- 引用块 Enter 续项、Tab 缩进或新的键盘命令
- 新的 Markdown 解析模型、AST、React 富文本层，或改写现有 main/preload 边界
- `TASK-014` 及之后的链接、图片、导出、搜索等 backlog 能力
- `TASK-022` 范围内更完整的跨平台 IME 行为扩展

Landing area:
- `src/renderer/code-editor.ts`
- `src/renderer/code-editor.test.ts`
- `src/renderer/styles.css`
- `docs/decision-log.md`
- `docs/test-report.md`
- `docs/progress.md`
- `docs/test-cases.md`
- `reports/task-summaries/TASK-013.md`

Acceptance:
- 非激活引用块显示为更接近渲染态的引用样式，`>` 前缀被弱化
- 光标进入引用块时，原始 Markdown 立即可见且可直接编辑
- 多行引用块在非激活态表现为同一块，而不是逐行散掉
- 引用块在点击、键盘移动、撤销重做后，源码态/渲染态切换不出现异常跳光标
- composition 期间不会提前触发破坏输入的装饰切换，composition 结束后只做一次派生状态 flush
- 不改变磁盘 Markdown 文本，不引入保存时重排

Verification:
- `npm run test -- src/renderer/code-editor.test.ts`
- `npm run lint`
- `npm run typecheck`
- `npm run test`
- `npm run build`
- 人工关注 `docs/test-cases.md` 中现有的 `TC-020 中文 IME`，并在需要时补充引用块场景

Risks:
- 引用块前缀是逐行语法，若装饰范围计算不准，容易破坏多行 blockquote 的光标映射
- 引用块样式如果改成 widget replacement 或 DOM 重排，容易踩中 P0 的 IME、undo/redo 和 round-trip 风险
- 失焦/聚焦与 active block 切换叠加时，可能出现装饰残留或闪烁
- blockquote 内部仍可能包含更复杂的 Markdown 结构，本轮必须保持范围收敛，避免过早扩张到嵌套语义
